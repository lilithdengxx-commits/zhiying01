# 🦅 智鹰 · ToG政企大客户AI商机挖掘系统
## 运行指南（压缩包版）

---

## 📋 前置要求

只需安装一个软件：**Node.js 20 LTS**

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 / 11（64位） |
| Node.js | >= 20.x LTS |
| 内存 | 建议 4GB 以上 |
| 网络 | 需要联网（用于安装依赖和实时抓取数据） |
| 端口 | 3001（后端）、5173（前端）需未被占用 |

---

## 🚀 第一次运行（三步完成）

### 第一步：安装 Node.js

1. 打开浏览器，访问 https://nodejs.org/
2. 点击 **"20.x.x LTS"** 下载 Windows 安装包（.msi）
3. 双击安装包，全部点击"Next"默认安装即可
4. 安装完成后，打开"命令提示符"（Win+R 输入 cmd），输入以下命令验证：

```
node -v
```

看到类似 `v20.x.x` 的版本号即为安装成功。

---

### 第二步：安装项目依赖

解压压缩包到任意目录（**建议英文路径**，如 `D:\zhiying`），然后：

- 双击 **`一键安装依赖.bat`**
- 等待窗口自动完成前后端依赖安装（约 2–5 分钟，取决于网速）
- 看到"全部依赖安装完成"提示后关闭窗口

> ⚠️ 此步骤只需要执行一次，后续启动无需重复安装。

---

### 第三步：启动系统

- 双击 **`一键启动全部.bat`**
- 脚本会自动打开两个窗口分别运行后端和前端
- 约 5 秒后浏览器将自动打开 **http://localhost:5173**

看到"智鹰"系统界面即启动成功 🎉

## ❓ 常见问题

### Q：双击 bat 文件一闪而过，什么都没发生？
**A：** 右键点击 bat 文件 → 选择"以管理员身份运行"。

---

### Q：浏览器打开了但页面空白或报错？
**A：** 检查后端是否已启动成功：
1. 打开浏览器访问 http://localhost:3001/api/health
2. 若返回 JSON 数据则后端正常，刷新前端页面即可
3. 若无法访问，请先确保"智鹰-后端API"窗口已打开且无红色报错

---

### Q：提示"端口已被占用"？
**A：** 3001 或 5173 端口被其他程序占用，关闭相关程序后重新启动；或重启电脑后再试。

---

### Q：依赖安装失败，提示网络错误？
**A：** npm 默认使用国外镜像，网络不稳定时可能失败。解决方法：
1. 打开命令提示符，运行以下命令切换为国内镜像：
```
npm config set registry https://registry.npmmirror.com
```
2. 然后重新双击"一键安装依赖.bat"

---

### Q：线索池/政策雷达数据为空？
**A：** 系统连接了政府采购网（ccgp.gov.cn）和新华网实时数据源，首次启动后约 30 秒完成第一次抓取。若长时间为空：
1. 检查网络是否可以访问 www.ccgp.gov.cn
2. 访问 http://localhost:3001/api/crawler/status 查看抓取状态
3. 访问 http://localhost:3001/api/crawler/run 手动触发一次抓取

## � 真假数据切换指南

系统支持两种运行模式，通过 `frontend/src/api.js` 第 6 行的一个开关控制。

### 模式对比

| | 模拟数据模式（Mock） | 真实数据模式（Real） |
|---|---|---|
| **开关值** | `const USE_MOCK = true` | `const USE_MOCK = false` |
| **数据来源** | `frontend/src/mockData.js`（本地固定数据） | 后端 `http://localhost:3001` 实时接口 |
| **需要启动后端** | ❌ 不需要 | ✅ 必须先启动后端 |
| **需要联网** | ❌ 不需要 | ✅ 需要（后端实时抓取外部数据） |
| **线索条数** | 固定 21 条 | 由实时抓取结果决定 |
| **适合场景** | 演示、离线展示、UI调试 | 接入真实政采/新华网数据 |

---

### 切换为真实数据模式（Mock → Real）

**第一步：修改开关**

打开 `frontend/src/api.js`，将第 6 行改为：
```js
const USE_MOCK = false
```

**第二步：启动后端**（必须，否则页面所有数据请求都会报错）

双击 `启动后端.bat`，等待窗口出现：
```
✅ Server running on port 3001
```

**第三步：重启前端**

重启前端 dev server（关闭后重新双击 `启动前端.bat`），  
然后访问 **http://localhost:5173**

> ⚠️ 如果只改了开关但没有重启前端，Vite 的热更新会自动生效，但建议重启一次以确保代理配置正常工作。

---

### 切换回模拟数据模式（Real → Mock）

**只需一步**：将 `frontend/src/api.js` 第 6 行改回：
```js
const USE_MOCK = true
```

Vite 热更新会立即生效，无需重启前端，**也无需后端运行**。

---

### 为什么真实模式必须启动后端？

`frontend/vite.config.js` 中配置了请求代理：
```
所有 /api/* 请求  →  转发到 http://localhost:3001
```
当 `USE_MOCK = false` 时，前端的每一个数据请求都会通过这条代理转发到后端。如果后端没有运行，所有接口都会返回 502 错误，页面数据全部为空。

---

### 快速自检

切换模式后，可以通过以下方式确认当前模式是否生效：

| 检查项 | 模拟模式 | 真实模式 |
|---|---|---|
| 线索池数据条数 | 固定 21 条 | 数量不固定 |
| 访问健康检查接口 | 不可访问（后端未启动） | 返回 JSON：`{"status":"ok"}` |
| 爬虫状态接口 | 不可访问 | 返回抓取计数和时间 |

> 健康检查地址：http://localhost:3001/api/health  
> 爬虫状态地址：http://localhost:3001/api/crawler/status
